什么是RPC

在分布式系统中，Remote Procedure Call可以理解为一台计算机的程序调用不同地址空间（通常是在相同网络环境下的其他计算机上）的另一台计算机的程序，在编码的过程中，就像在本地调用一样，不需要程序员明确的编码远程交互的细节。

Remote Procedure Call（RPC）是一种协议，一个程序可以使用该协议从位于网络上的另一台计算机中的程序请求服务，无需了解网络的详细信息。

过程调用一般也称为函数调用或者子例程调用。

RPC的执行属于Client-Server模型，调用的一方作为Client，提供服务的一方作为Server。和常规的本地函数调用一样，调用发起后，调用方（client）会处于阻塞的状态，直到被调用的Server方返回计算的结果。在多线程的环境下，可以同时发起多个RPC调用。远程调用相对于本地调用来讲，速度会慢几个数量级。

RPC其实属于进程间通信（IPC，inter-process communication）的一种形式，不同的进程有不同的地址空间，比如程序处于不同的机器，或者程序位于同一机器对外暴露不同的端口。

消息传递

RPC采用了请求-响应机制，由客户端向已知的远程服务器发送请求，并附带参数，使得远程的服务器执行既定的过程，然后远程的服务端在计算完毕后，响应客户端的请求，返回计算结果给客户端。当服务端在执行计算的时候，客户端处于阻塞状态的，它在等待服务端的计算结果，除非客户端发送了异步的请求给服务器，比如采用了XMLHttpRequest。

对于远程过程调用来讲，一个最重要的不同就是因为不可预测的网络问题导致调用失败。导致调用失败的因素有很多，可能是因为网络问题，或者是服务提供者服务挂掉等。

调用过程

  1. 客户端调用client stub，这次调用为本地调用，同时传入参数
  2. client stub将参数打包到消息中，进行系统调用发送消息。打包参数称为序列化/编组
  3. 客户端的本地计算机操作系统通过Socket将消息发送到服务端的计算机。
  4. 服务端Socket在接收到消息后，将数据包传入server stub。
  5. server stub解压缩消息中的参数，解包称为反序列化/解组。
  6. server stub调用服务器上的过程，将计算结果返回给客户端。
